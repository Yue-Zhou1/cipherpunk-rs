2. Missing point validation — small subgroup attack
// BUG: accepting external G1/G2 points without subgroup check
fn bad_verify(pk: G1Affine, msg: &[u8], sig: G2Affine) -> bool {
    // no check that pk is in the correct r-torsion subgroup
    // attacker supplies small-subgroup point → pairing result is predictable → forgery
    pairing(pk, hash_to_g2(msg)) == pairing(g1_gen(), sig)
}

// FIX: explicit subgroup membership check before use
fn good_verify(pk: G1Affine, msg: &[u8], sig: G2Affine) -> bool {
    assert!(pk.is_on_curve() && pk.is_in_correct_subgroup_assuming_on_curve());
    assert!(sig.is_on_curve() && sig.is_in_correct_subgroup_assuming_on_curve());
    pairing(pk, hash_to_g2(msg)) == pairing(g1_gen(), sig)
}

3. Nonce reuse — private key recovery
// BUG: deterministic nonce not bound to the message
fn bad_sign(sk: &Scalar, msg: &[u8]) -> Signature {
    let k = hash_to_scalar(sk.to_bytes());  // nonce independent of msg
    // two different messages → same k → two equations, two unknowns → sk recovered
    schnorr_sign_with_nonce(sk, msg, k)
}

// FIX: RFC 6979 / BIP340 — nonce must bind both sk AND msg
fn good_sign(sk: &Scalar, msg: &[u8]) -> Signature {
    let k = rfc6979_nonce(sk, msg);  // HMAC-DRBG(sk || msg)
    schnorr_sign_with_nonce(sk, msg, k)
}

Use these tools in combination — no single tool covers all categories.
Static analysis
cargo audit — checks all dependencies against the RustSec CVE advisory database
cargo clippy -- -D warnings — lint with focus on clippy::integer_arithmetic, clippy::as_conversions, clippy::unwrap_used
cargo geiger — counts unsafe usage per crate, generates an unsafe density report across the dependency tree
cargo deny — license audit + dependency tree inspection, surfaces hidden transitive dependencies
rudra — static unsafe memory safety analyser, detects panic safety violations and unsound Send/Sync impls
mirai (Meta) — abstract interpretation framework, detects integer overflow and reachable unwrap panics
Dynamic analysis and fuzzing
cargo-fuzz (libFuzzer) — fuzz deserialization entrypoints, point decompression, and proof verification paths
honggfuzz-rs — coverage-guided fuzzing, often faster than libFuzzer on certain input shapes
proptest / bolero — property-based testing to verify algebraic laws (associativity, inverses, homomorphism)
Differential fuzzing: feed identical inputs to two implementations (e.g. two SHA-256 precompiles) and compare outputs byte-for-byte
cargo-valgrind / Miri — detect reads from uninitialised memory and memory leaks in unsafe code paths
Side-channel detection
dudect (Rust port) — statistical timing analysis using Welch's t-test to verify constant-time properties empirically
ctgrind / Valgrind memcheck — mark secret bytes as "uninitialised", then track whether any conditional branch or memory address depends on them
LLVM IR inspection — compile with --emit=llvm-ir and grep for secret-dependent br instructions or variable-address memory accesses
subtle crate audit — confirm all secret-value comparisons use ct_eq() not ==, and all conditional selections use ConditionallySelectable
Supply chain and dependencies
cargo tree — full dependency tree; identify cryptographic library versions in transitive deps that differ from direct deps
Check whether Cargo.lock is committed — an uncommitted lock file means builds are not reproducible
Identify git = "..." dependencies — these bypass crates.io auditing and may point to unreviewed forks
Inspect feature flags for hidden unsafe entrypoints: cfg(feature = "unsafe_internals"), cfg(test), debug-only code paths that expose internals
